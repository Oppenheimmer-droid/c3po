BEFORE DEPLOYMENT:
  ☐ Generate SECRET_KEY
  ☐ Get OpenAI API key
  ☐ Create Railway account (https://railway.app)
  ☐ Create Vercel account (https://vercel.com)
  ☐ Authorize GitHub access for both

RAILWAY DEPLOYMENT:
  ☐ Create New Project → Deploy from GitHub
  ☐ Select repository: c3po
  ☐ Set root directory: backend
  ☐ Create PostgreSQL database
  ☐ Create Redis cache
  ☐ Configure environment variables
  ☐ Verify deployment complete
  ☐ Note the public URL

VERCEL DEPLOYMENT:
  ☐ Create New Project → Import from GitHub
  ☐ Select repository: c3po
  ☐ Set root directory: frontend
  ☐ Configure NEXT_PUBLIC_API_URL (Railway URL)
  ☐ Verify deployment complete
  ☐ Note the public URL

FINAL CONFIGURATION:
  ☐ Update CORS_ORIGINS in Railway (set to Vercel URL)
  ☐ Test login flow on frontend
  ☐ Verify API connectivity
  ☐ Test document upload
  ☐ Check analytics dashboard


🔒 ENVIRONMENT VARIABLES (Railway)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SECURITY:
  SECRET_KEY=<generated-key>
  ENVIRONMENT=production
  DEBUG=false

APIs:
  OPENAI_API_KEY=<your-key>

DATABASE (auto-configured):
  DATABASE_URL=postgresql+psycopg2://...
  POSTGRES_USER=<auto>
  POSTGRES_PASSWORD=<auto>

CACHE (auto-configured):
  REDIS_URL=redis://...
  CELERY_BROKER_URL=${REDIS_URL}/1
  CELERY_RESULT_BACKEND=${REDIS_URL}/2

STORAGE:
  CHROMA_PERSIST_DIR=/app/chroma_data

API:
  API_HOST=0.0.0.0
  API_PORT=8000
  CORS_ORIGINS=https://your-vercel-domain.vercel.app

🆘 TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Issue: Backend won't start
→ Check: Railway logs → search for errors
→ Verify: DATABASE_URL and REDIS_URL are set
→ Check: alembic upgrade head succeeds

Issue: Frontend can't connect to backend
→ Check: NEXT_PUBLIC_API_URL is correct
→ Check: CORS_ORIGINS in Railway includes frontend URL
→ Check: DevTools Network tab for failed requests

Issue: Database empty
→ Check: Run migration manually in Railway terminal
→ Run: alembic upgrade head

See DEPLOY_STEPS.md "Troubleshooting" section for more
